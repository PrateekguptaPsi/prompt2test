Root-Cause Analysis

Systemic root cause: the healthplix.* custom models ship without server-side constraints (Odoo @api.constrains / SQL CHECK) on clinically and financially critical numeric fields. All four confirmed defects reproduce via direct JSON-RPC create calls, proving the ORM layer accepts the invalid values; any validation that exists lives only in UI widgets and is bypassable by any API client, integration, or import job.

DefectFieldMissing guardRoot cause
BUG-ONCO-001 (KAN-49)healthplix.prescription.line.quantity quantity > 0 constraint No @api.constrains('quantity'); field defined as plain Float with no domain.
BUG-ONCO-004 (KAN-52)healthplix.prescription.line dosage text Dosage plausibility / vocabulary check Free-text Char field; no validation list or range parsing for dosage strings.
BUG-ONCO-002 (KAN-50)healthplix.billing.line.price price >= 0 constraint No non-negative constraint; negative unit price flows into computed subtotal and invoice total.
BUG-ONCO-003 (KAN-51)healthplix.doctor.experience experience >= 0 constraint Plain Integer field; no bounds validation at model level.

Why severity is patient-safety led: a negative or implausible medication quantity/dosage that persists to the prescription record can propagate to pharmacy fulfilment and administration records in an oncology context, where dosing errors are life-threatening.

Recommended fix pattern: add model-level @api.constrains for each field above (single small patch per model), plus a regression re-run of the API boundary bug hunter in 05_automation/tests/api-business-rules.spec.ts, which will turn green automatically once the constraints exist — no test changes needed.

Enhancements (Non-defect observations)

  • Add centralized server-side numeric constraint policy across healthplix.* models for non-negative and bounded clinical/financial fields.
  • Introduce dosage validation rules (range, pattern, and unit constraints) with explicit fail-safe review actions.
  • Add regression API checks in CI for all boundary constraints currently represented by confirmed defects.
  • Enable secondary synthetic user credentials in automation environment to continuously execute cross-user isolation probes.
